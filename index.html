Gravitationally Lensed Quasar System Analysis

Dec. 2020 - Present
Skills used: Python (pandas, NumPy, astropy, matplotlib, pickle), spectral analysis, LaTeX, QFitsView




Gravitational lensing occurs when a large galaxy bends light emitted from objects behind that galaxy. This astronomical phenomenon acts as a cosmic telescope and creates distorted and/or multiple images (also called sightlines) of the background objects. Gravitationally lensed quasar light can provide information about the circumgalactic medium (CGM) of foreground galaxies at different sightlines. The CGM is the large reservoir of gas that surrounds a galaxy, and it is important for understanding galaxy evolution.

For this project, I examined a quadruply lensed quasar system called J1004+4112. It has 4 sightlines, A, B, C, and D, as seen as in the image below:

There are three galaxies, G1, G2, and G3, that make up a galaxy cluster at redshift (z) 0.68. This cluster is causing the gravitational lensing of a quasar behind the galaxy cluster at z = 1.734. To put 'redshift' into context, objects at higher redshifts are farther away from Earth. I used Keck Cosmic Web Imager (KCWI) integral field unit (IFU) observations of J1004+4112. KCWI is a spectrograph located on Keck, a 10-meter telescope in Hawaii. The IFU instrument produces a 3D data cube where each image pixel contains a spectrum or flux versus wavelength graph.

I used Python to extract a spectrum from each of the four quasar images. I then identified absorption lines in each spectrum using a package called linetools. Absorption lines are dips in flux that indicate the presence of diffused gas. The figure below shows all four image spectra and identified absorption lines that are denoted by short vertical lines above each spectrum:

Each absorption line is color-coded by the redshift it was found at (called an absorber redshift).

The following figure gives a zoomed-in look at absorption lines at the absorber redshift 0.676:

Each spectrum is color-coded by its corresponding image. One interesting observation is that Mg II 2796 absorption is seen in images A, B, and D at z = 0.676. This absorber redshift is very close to the galaxy cluster redshift (z = 0.68) which suggests that there is cool gas in the galaxy cluster halo. This is rare since most galaxy cluster halos typically lack cool gas.

Additionally, it is clear to see that there are variations in absorption line strengths across the four images. For example, at z = 0.676, sightline D has a visibly strong Fe II 2586 absorption line while the other three sightlines do not. These variations hold information about the CGM gas. To analyze these variations, I used Python to calculate equivalent widths, which measure absorption line strength, for each absorption line. I then computed fractional differences in equivalent width and plotted these against physical separation:

This figure shows Mg II and Fe II fractional differences for J1004+4112 (opaque data points) and other quasar systems (transparent data points). This plot tells us that the spatial extent of the absorbers can be constrained to < 45 kiloparsecs.

Currently, I am writing a research paper to publish these findings. To read more about this research, please view this interactive poster that I presented at the American Astronomical Society meeting in Pasadena, California in June 2022.

Slices of Life

Sep. 2023 - Dec. 2023
Skills used: user interface development, Python, PyQt, leadership




In the fields of biology and biophysics, understanding the internal structure and structural function of a biological sample is important. Biologists and biophysicists learn this by using a process called serial electron microscopy. The first step of this process is to encase the biological sample in resin. Next, the resin-encased sample is cut into 50-100 nanometer thin slices using a microtome. After the slices are cut, they float in a small "water reservoir" and are then carefully collected onto grids. The grids with the slices are then inserted into a transmission electron microscope for imaging. Each slice is a cross-section of the biological sample and thus shows the internal structure. If the order of the slices were kept, a 3D reconstruction of the biological sample can be created with the images taken with the electron microscope.

This process is incredibly tedious and time-consuming for researchers. For our senior design project, my team and I were tasked with finding a way to improve the current serial electron microscopy workflow. We worked on a user interface and tracking algorithm that tracks the biological sample slices as they come off the microtome and float in the water reservoir. As the team lead, I facilitated team meetings and discussions. I took the lead on developing and testing the user interface.

Our final project presentation slide deck, which we presented on at the end of the semester to the physics department, can be viewed here. I recommend viewing it in presentation mode. American Bar Association Client Dashboard

The dataset came from an ABA website that offers pro bono services to clients. The website allows clients to ask questions to attorneys. My team and I decided to create an interactive client dashboard using Tableau to investigate the typical client profiles for specific question categories (such as family, homelessness, etc.). One interesting insight we found is that for the housing and homelessness question category, the typical client profile is a single, young female in financial need from New York, Florida, or Texas. Electric Vehicle Charging Station Locations Model

May 2022 - Aug. 2022
Skills used: Python (pandas, NumPy, geopandas, matplotlib), SAS Model Studio, SAS Visual Analytics




One of my projects as a SAS intern was to help develop a model that will be used in a customer-facing demonstration to find optimal electric vehicle charging station locations in North Carolina. First, I built a "feature dataset" to feed into the model. To create this dataset, I used Python (pandas, NumPy, geopandas, and matplotlib libraries) to compile various datasets on the North Carolina interstate network. These datasets came from the NC Department of Transportation, the NC Office of State Budget and Management, and the US Department of Energy. The data from this dataset is used by the model to determine which parts of the NC interstate network need charging stations. Each row of the dataset is a road segment of the interstate network. The dataset contained information such as the unique numeric ID for each road segment, the segment length, the annual average daily traffic in 2019 in number of vehicles, the number of cities near the road segment, the number of charging stations near the segment, and a weightage variable that indicates the relative installation cost for a charging station near the segment. The installation cost weight variable is based on the number of cities near a road segment. The greater the number of cities near a segment, the higher the installation cost weight value and thus, the lower the cost for installing a charging station at that location. This is because areas near cities tend to have most of the infrastructure already in place for the incorporation of a new charging station.

To create the training dataset, I selected 100 random road segments from the feature dataset. My team at SAS and I determined which of the road segments needed a charging station by adding a "Needs Station" column and manually entering a '1' for road segments that should have a charging station and '0' for segments that did not.

Next, I used SAS Visual Analytics and SAS Model Studio to create a multiple logistic regression model that determines which road segments from the testing dataset require a charging station. The model turned out to be approximately 80% accurate. About 37% of the interstate segments in the testing dataset require a charging station.

The next step for this project is to plot all the road segments in the feature dataset on a map of North Carolina and color code the segments that require a charging station by the installation cost weight variable. By using this map, a potential energy customer can make well-informed decisions about where to place charging stations. A few additional next steps are to improve the model by adding more variables to the feature dataset and to try other types of models (such as an optimization model) and compare model outputs and accuracies.
